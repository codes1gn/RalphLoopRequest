CUDA kernels (.cu) and compiler flags used in this repo

Scope: `.cu` files appear throughout `kernels/`, `tuning*/`, shape directories, sweeper templates, etc. Flags are not embedded in `.cu` sources; they come from `kernels/manifest.json` and Python/shell build drivers.

Canonical flags (kernels/manifest.json → build)
- nvcc_flags string: `-arch sm_90a -std=c++17 -DCUTLASS_ENABLE_TENSOR_CORE_MMA=1 -D__CHOREO_TARGET_CUTE__ -D__USE_CUDA_TYPE__ -D__CHOREO_DMA_DIAGNOSIS__ -Xcompiler -static-libstdc++ -O2 --use_fast_math`
- nvcc_includes: `-I${CHOREO_REPO}/runtime -I${CHOREO_REPO}/extern/cutlass/include -I${CHOREO_REPO}`
- nvcc_libs: `-L/usr/local/cuda/lib64 -lcuda`
- Dtype extras: `f16_extra`: `-ftz=true`; `e4m3_extra`: (empty)
- choreo_flags (for .co pipeline): `-gs -t cute -arch=sm_90a --use-warpspec --use-prepack`

Variants seen in tuning_v2/tuning scripts (unique / notable)
- Architecture: `-arch sm_90a` OR `-gencode arch=compute_90a,code=sm_90a` (equivalent targeting pattern in list form).
- C++ standard: `-std=c++17` (most scripts); skills sometimes show `-std=c++20` for example invocations.
- Optimization: `-O2` (common) vs `-O3` (some f16 bank / variant scripts).
- Math: `--use_fast_math`; f16 often adds `-ftz=true` inline or via extra flags.
- Defines: `DCUTLASS_ENABLE_TENSOR_CORE_MMA=1`; optional `-DSPMM_DTYPE_E4M3` for e4m3; shape compile `-DSPMM_DEFAULT_M/N/K` (from skills and build docs).
- Choreo-specific defines in some sweeper compile scripts: `-D__CHOREO_TARGET_CUTE__ -D__USE_CUDA_TYPE__ -D__CHOREO_DMA_DIAGNOSIS__` (matches manifest nvcc_flags).
- Linker passthrough: `-Xcompiler -static-libstdc++`
- Includes/libs: same CHOREO + Cutlass include trio and `-lcuda` pattern as manifest.

Per-shape extras
- `meta.json` fields `extra_nvcc_flags` (often empty string) append to the base nvcc invocation in bank compile scripts.

Summary set (deduplicated concepts)
- `-arch sm_90a` / `-gencode arch=compute_90a,code=sm_90a`
- `-std=c++17` (and occasional `-std=c++20` in examples)
- `-O2`, `-O3`
- `--use_fast_math`, `-ftz=true`
- `-DCUTLASS_ENABLE_TENSOR_CORE_MMA=1`, `-D__CHOREO_TARGET_CUTE__`, `-D__USE_CUDA_TYPE__`, `-D__CHOREO_DMA_DIAGNOSIS__`, `-DSPMM_DTYPE_E4M3`, `-DSPMM_DEFAULT_*`
- `-Xcompiler -static-libstdc++`
- `-I…/runtime`, `-I…/extern/cutlass/include`, `-I…/choreo_repo`
- `-L/usr/local/cuda/lib64 -lcuda`
- Choreo CLI: `-gs -t cute -arch=sm_90a --use-warpspec --use-prepack` (and variants with `-es` for emit in some scripts)
