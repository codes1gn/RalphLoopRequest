Task: Makefile and build process (control_11)

## Root Makefile

Location: `Makefile` at repository root.

Variables:
- `PAPER_DIR := paper`
- `FIG_DIR := figures`

Targets:

- **all** (default) — Depends on `paper`.
- **svg2pdf** — For each `figures/out/*.svg`, converts to PDF with `cairosvg` if the SVG is newer than the PDF or the PDF is missing.
- **paper** — Depends on `svg2pdf`. Runs `tectonic main.tex` inside `paper/`. Then runs an inline Python one-liner using `PyPDF2` to filter PDF pages (drops near-empty or boilerplate pages based on extracted text). Failures in the PDF filter are ignored (`2>/dev/null || true`).
- **figures** — `cd figures && npm run build` (Node/D3 figure pipeline per `figures/package.json`).
- **preview** — Depends on `paper`; uses `pdftoppm` to rasterize `paper/main.pdf` into PNGs under `paper/preview/`.
- **clean** — Removes LaTeX auxiliary files under `paper/` and clears `figures/output/`.
- **watch** — `tectonic -X watch main.tex` in `paper/`.
- **snapshot** — `bash scripts/snapshot.sh "$(or $(MSG),draft)"` — note: `scripts/snapshot.sh` may be absent in some checkouts; if missing, this target will fail until the script exists.

## Prerequisites (implicit)

- `tectonic` for LaTeX
- `cairosvg` for SVG→PDF
- Python 3 with `PyPDF2` for the optional PDF post-process
- `pdftoppm` (poppler) for `preview`
- Node/npm for `figures` build

## CUDA / kernel builds

The root Makefile does **not** build GPU kernels. CUDA/nvcc compilation is driven by scripts under `tuning/`, `tuning_v2/scripts/`, and related shell scripts (see control_13), typically invoking `nvcc` with includes pointing at a separate **choreo** checkout.

## Summary

The documented “build” for this repo is **paper + figures**: SVG preprocessing, LaTeX compilation with Tectonic, optional PDF cleanup, and an npm-based figure generator. GPU tuning binaries are built outside this Makefile via tuning scripts.
