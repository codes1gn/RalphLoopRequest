Task: .claude/skills/ tuning strategies — comparison (control_17)

## Shared infrastructure

Most tuning skills delegate to **CroqTuner FSM** (`.claude/skills/fsm-engine/`): `pre-step-check.sh` → step → `post-step-check.sh` → `state-transition.sh`, with **guard flags**, **idea logs**, and **checkpointed state JSON**. They also reference **`kernels/manifest.json`**, **`tuning/state.json`**, and the **choreo** program rules.

---

## ai-tune-from-current-best

- **Goal:** Adapt the **reference best** `.cu` (f16 iter143, e4m3 iter068) to **every** scenario shape.
- **Budget:** Up to **30 iterations per shape**; **non-stop sweep** until all shapes processed (DFS-style depth on each shape).
- **State file:** Default **`loop-state.json`**.
- **Regions:** Prioritizes **near** reference `(4096,8192,8192)` shapes; uses manifest scenarios (134 unique shapes per dtype after dedup).
- **Artifacts:** Keys `<dtype>_<M>x<N>x<K>` under `tuning/logs|srcs|perf|checkpoints/`.

---

## ai-tune-from-scratch

- **Goal:** Tune **far** / typical shapes from **baseline `.co`** kernels (minimal correct kernels), discovering structure independently.
- **Budget:** **≥150 iterations** per shape (deep search).
- **State file:** **`loop-state_from_scratch.json`** (separate file so concurrent sessions do not clash).
- **Artifact keys:** **`_fs` suffix** (e.g. `f16_768x768x768_fs`) to isolate from from-current-best runs.
- **Profiling:** **ncu mandatory** each iteration (per skill emphasis); stronger requirement for evidence than quick adapt.
- **Use case:** Shapes where reference topology is a poor match (small squares, extreme aspect ratios, etc.).

---

## ai-tune-bfs (breadth-first)

- **Same goal family as from-current-best** (adapt reference best to all shapes).
- **Difference:** **Early exit on first KEEP** (or exhaust 30 iters) — visits every shape for **baseline + first win** before any shape gets a second deep pass; **coverage-first**.
- **Same default FSM state** as DFS from-current-best (`loop-state.json`), same storage layout.
- **Use case:** Rapid coverage of the landscape; not maximizing per-shape TFLOPS in one pass.

---

## push-marginal

- **Goal:** Improve **18 specific e4m3 shapes** that **lose or barely beat cuSPARSELt**.
- **State file:** **`push-marginal-state.json`**.
- **Phases:** **Phase 1** — up to **50** choreo DSL iterations from baseline `.co` / variants; **Phase 2** — automated **sweeper** ideas on best Phase 1 result.
- **Queue:** **`tuning/push_marginal_queue.json`**.
- **Use case:** Targeted recovery on worst/marginal shapes, not the full 134-shape sweep.

---

## iterative-improve

- **Goal:** **Infinite-style** improvement of **verified kernel bank** entries (not the full scenario sweep).
- **State file:** **`iterative-improve-state.json`**.
- **Selection:** Picks shapes with **lowest speedup vs cuSPARSELt** (most headroom).
- **Flow:** SELECT_SHAPE → CHOREO phases → NCU → GENERATE_CU → SWEEPER → COMPARE → UPDATE_BANK → repeat.
- **Use case:** Bank polishing after initial coverage.

---

## fsm-engine (not a tuning strategy by itself)

- **Router + mechanics** for all of the above: legal transitions, INIT, guard enforcement, compaction protocols.

## ralph-loop-request

- **Meta-skill / continuation gate** for agent tasks — not a GPU tuning algorithm.

## Comparison table

| Skill | Primary seed | Iters/shape | State file | Termination emphasis |
|-------|----------------|------------|------------|----------------------|
| from-current-best | Reference best .cu | 30 (full) | loop-state.json | Complete 30 per shape |
| from-scratch | Baseline .co | 150+ | loop-state_from_scratch.json | Deep until max_iter |
| ai-tune-bfs | Reference best .cu | ≤30, often stops early | loop-state.json | First KEEP or exhaust |
| push-marginal | Baseline / variants | 50 + sweeper phase | push-marginal-state.json | 18-shape queue done |
| iterative-improve | Bank kernel | open-ended | iterative-improve-state.json | Bank-wide improvement |

**Summary:** Strategies split along **seed type** (reference .cu vs baseline .co), **coverage vs depth** (BFS vs DFS), **scope** (all scenario shapes vs 18 marginal vs bank-only), and **parallelism of state files** (scratch vs current-best vs marginal).
