Summary: kernels/manifest.json schema (version 3)

Top-level keys:
- version (number): schema/registry version (3).
- description (string): human-readable summary of the CrokTile kernel registry.
- hardware (object): target GPU facts — gpu name, sm architecture string, cuda/driver versions, peak TFLOPS (f16/f8), SM count, smem per SM, L2 size.
- choreo_repo, gemm_sp_paper_repo, gemm_sp_kernel_dir (strings): absolute paths to related repos/dirs.
- reference_kernels (object): per-dtype (e.g. f16, e4m3) entries with best_cu path, best_tflops, best_hw_eff, tuned_shape [M,N,K], origin_iter, origin_readme.
- baseline_kernels (object): per-dtype filenames for canonical baseline .co; includes _note.
- scenarios (object): named scenario configs. Each scenario has label, type ("single" | "tied_pair"), and scenario-specific fields:
  - single: M, N, K may be numeric or "S" (swept), plus sizes[].
  - tied_pair: tied_dims, free_dim, sub_series (nested objects with label, fixed dims, tied_sizes or vary_sizes).
- size_constraints (object): min_M, min_N, min_K.
- region_clustering (object): _note, near_threshold / far_threshold (descriptive strings), typical_shapes_for_scratch (per-dtype list of [M,N,K] triples).
- build (object): nvcc_flags, nvcc_includes, nvcc_libs (string templates with env placeholders), f16_extra, e4m3_extra, choreo_flags, warmup, iters.
- verification (object): per-dtype base_tol and rel_tol.
- tuning_limits (object): from_current_best and from_scratch each with max_iter and description.

Notes: Paths under reference_kernels are absolute; build strings embed ${CHOREO_REPO} in includes. Scenario names cover square, sweep_m/n/k, and tied sweeps (sweep_mn, sweep_mk, sweep_nk) with sub_series for fixed-dimension variants.
