Project overview (directory structure)

Repository role
- croktile_paper is the academic paper and experiment workspace for “CrokTile”: a compiler-guided auto-tuning story for structured sparse GEMM, paired with extensive kernel tuning artifacts (Hopper-class GPUs, SM90).

Top-level layout
- paper/ — LaTeX source (ACM-style paper), references.
- figures/ — D3.js figure generators (figures/src), generated SVG outputs (figures/out), Node dependencies under figures/node_modules.
- ae/ — artifact evaluation materials (logs, data for reproducibility).
- kernels/ — Kernel registry (manifest.json) and per-dtype curated best kernels (e.g. gemm_sp_f16/, gemm_sp_e4m3/).
- tuning/, tuning_v2/, tuning_v3/ — successive generations of tuning infrastructure: logs, kernel banks, sweeper variants, plotting.
- scripts/ — small generators for shape suites and state.
- .claude/skills/ — agent skills (FSM tuning engine, ai-tune modes, iterative improve, etc.).
- .cursor/rules/ — Cursor rules for perf-tuning dispatcher, workspace policy, paper style, figures, methodology.
- literature-review.md, RESEARCH-PLAN.md — research notes and plan.

Unusual layout: shape-prefixed directories at repo root (e.g. 4096x8192x8192/, 768x768x768/) holding per-shape build artifacts (e.g. compiled `kernel` binaries, logs) alongside the hierarchical copies under tuning*/kernel_bank/. This makes the tree wide and artifact-heavy.

Supporting docs
- Makefile — paper/build convenience targets.
- .gitignore — excludes generated/binary content as configured.

In short: the project mixes LaTeX/paper assets, visualization, and a large body of GPU tuning scripts and stored kernels organized by shape and dtype.
