Task 13 — Shell scripts in the project (purpose of each)

Inventory (11 `.sh` files under repo, excluding nested tool caches)
--------------------------------------------------------------------

1. `tuning/kernel_bank/compile_all.sh`
   Compiles all `kernel.cu` under `tuning/kernel_bank/e4m3/<shape>/` to a `kernel` binary per folder. Reads `extra_nvcc_flags` from each `meta.json`. Uses `CHOREO_REPO=/home/albert/workspace/choreo`, nvcc sm_90a, optional `--parallel N`.

2. `tuning/kernel_bank/run_all.sh`
   Runs each compiled `kernel` in the e4m3 bank: correctness check, then timed measurement; updates each `meta.json` and rebuilds `registry_e4m3.tsv` and `results_e4m3.json`. Options: `--gpu`, `--shapes`.

3. `tuning/kernel_bank/run_f16_bank.sh`
   Same pattern for `tuning/kernel_bank/f16/`: verify, measure, HW efficiency vs peak, writes `registry_f16.tsv`.

4. `tuning_v2/scripts/compile_all.sh`
   Generalized compile-all for `tuning_v2/kernel_bank/<dtype>/<shape>/`. Supports `--dtype e4m3|f16|all`, `--parallel`, `--shapes`.

5. `tuning_v2/scripts/run_all.sh`
   Generalized run-all for both dtypes under `tuning_v2/kernel_bank/`, writes per-dtype `registry_*.tsv` and `results_*.json`.

6. `tuning_v2/scripts/compile_sweeper.sh`
   Compiles sweeper variant `.cu` files from `tuning/sweeper_srcs/<dtype>/` into `tuning_v2/sweeper/<dtype>/bins/`. Args: dtype (e4m3|f16|all), job count.

7. `.claude/skills/fsm-engine/scripts/pre-step-check.sh`
   CroqTuner FSM guard: validates preconditions for a named step (`BASELINE`, `IDEATE`, …) using `jq` on `loop-state.json` (or `CROQTUNER_STATE_FILE`). Exits non-zero if step should be blocked.

8. `.claude/skills/fsm-engine/scripts/post-step-check.sh`
   Validates postconditions after a step (directories, flags, logged results). Complements `pre-step-check.sh`.

9. `.claude/skills/fsm-engine/scripts/state-transition.sh`
   Legal FSM transitions; updates `loop-state.json` atomically with `INIT` and key=value overrides; resets guard flags per transition.

10. `ralph-loop-request-data/s1/treatment_02.sh`
    Experiment harness: find duplicate files by MD5 under a directory (treatment variant using `uniq`).

11. `ralph-loop-request-data/s1/control_02.sh`
    Experiment harness: duplicate-file finder (control variant using `awk` grouping).

Makefile reference: root `Makefile` invokes `bash scripts/snapshot.sh` for `snapshot` target — script not verified present in repo snapshot.
