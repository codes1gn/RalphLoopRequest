Project directory overview — croktile_paper

Purpose: Academic paper + artifact repository for “CrokTile” — compiler-guided auto-tuning of structured sparse GEMM kernels, with convergence/search-space evidence distinct from a companion kernel-only paper (see research-methodology rule).

Major subtrees:
- paper/ — LaTeX source (ACM sigconf), references.
- figures/ — D3.js sources under figures/src/, generated SVG under figures/out/, Node tooling (package.json); publication figures.
- kernels/ — manifest.json (single source of truth for scenarios, build, references), gemm_sp_f16/ and gemm_sp_e4m3/ curated best kernels per shape.
- tuning/ — legacy tuning logs, srcs, perf, checkpoints, kernel_bank, sweeper logs, BFS/autotune scripts, comparisons to cuSPARSELt.
- tuning_v2/ — newer sweeper templates, kernel_bank tree (large per-shape layout), scripts for compile/measure/bank maintenance.
- tuning_v3/ — plotting and data collection for paper figures.
- ae/ — artifact evaluation materials (data, scripts).
- scripts/ — small generators (state, shape suites).
- .claude/skills/ — agent skills (FSM tuning engine, ai-tune modes, iterative improve, etc.).
- .cursor/rules/ — Cursor rules for perf-tune, tuning workspace, paper style, figures, methodology.

Repository root also contains many directories named like MxNxK (e.g. 4096x8192x8192) — shape-key artifact folders (compiled artifacts / logs tied to specific GEMM shapes), alongside first-class dirs above.

Technology mix: CUDA (.cu/.co), Python automation, shell helpers, JSON/TSV logs, LaTeX/BibTeX, JavaScript (figures).
