Task 16 — Hardcoded paths in the codebase (representative list)

Method
------
Searched for absolute path patterns (`/home/`, legacy `/Users/`, Windows-style) in source-like files. The repo embeds **machine-specific** paths for the developer’s layout; many generated kernels repeat the same choreo path in **comments** (provenance from choreo compiler).

Categories
----------

A. **Canonical workspace roots (configuration)**
- `/home/albert/workspace/choreo` — choreo compiler/runtime/Cutlass includes (see `kernels/manifest.json` keys `choreo_repo`, `gemm_sp_kernel_dir`; repeated in shell scripts as `CHOREO_REPO`).
- `/home/albert/workspace/gemm_sp_paper` — `manifest.json` field `gemm_sp_paper_repo`.
- `/home/albert/workspace/croktile_paper` — used in `tuning_v3/collect_data.py`, `plot_options.py`, `plot_comparison.py` as `REPO = "..."`.

B. **Reference kernels and docs inside choreo** (manifest + skills)
- Paths under `/home/albert/workspace/choreo/benchmark/performance/gemm_sp/` to `best_cu`, README markdown, baseline `.co` files.

C. **CUDA installation**
- `/usr/local/cuda/lib64` and `/usr/local/cuda/bin/nvcc` in compile scripts (`compile_all.sh`, `compile_sweeper.sh`).

D. **Generated / copied kernel sources**
- Many `tuning_v2/kernel_bank/**/*.cu` and `tuning_v2/sweeper/f16_variant_templates/*.cu` contain **comments** with paths like `/home/albert/workspace-choreo/choreo-cursor/benchmark/...` (alternate machine naming) — provenance only, not active includes.

E. **Experiment / artifact JSON** (`ralph-loop-request-data/`, large dumps)
- Embedded absolute paths in JSON for `best_kernel`, `source_origin`, error strings referencing `/home/albert/workspace/choreo/runtime/choreo.h`, etc.

F. **Skills and rules**
- `.claude/skills/*.md` reference `/home/albert/workspace/choreo/.claude/program.md` and similar for the tuning loop protocol.

Portability note
----------------
Cloning the repo on another machine **requires** updating `kernels/manifest.json`, tuning_v3 `REPO` constants, and shell `CHOREO_REPO`, or using environment variables (not uniformly supported everywhere — many scripts hardcode).
