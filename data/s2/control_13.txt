Task: Shell scripts in the project and purpose (control_13)

Scope: `*.sh` tracked under the repo (excluding only listing experiment harness copies in `ralph-loop-request-data` where noted).

## tuning/kernel_bank/compile_all.sh

Compiles all `kernel.cu` under `tuning/kernel_bank/e4m3/<shape>/` to a `kernel` binary; reads **extra_nvcc flags** from each `meta.json`; optional `--parallel N`. Uses fixed **choreo** include path and `/usr/local/cuda/lib64`.

## tuning/kernel_bank/run_all.sh

Runs every compiled e4m3 bank binary: correctness pass then timed TFLOPS; updates each `meta.json` and rebuilds `registry_e4m3.tsv` and `results_e4m3.json`. Args: `--gpu`, `--shapes` list.

## tuning/kernel_bank/run_f16_bank.sh

Same pattern for `tuning/kernel_bank/f16/` — verify, measure, HW efficiency check vs f16 peak, writes `registry_f16.tsv`.

## tuning_v2/scripts/compile_all.sh

Generalized bank compile for **e4m3** and/or **f16** under `tuning_v2/kernel_bank/`; supports `--dtype`, `--parallel`, `--shapes`.

## tuning_v2/scripts/run_all.sh

Generalized run-all for both dtypes under `tuning_v2/kernel_bank/`; writes per-dtype registry and results JSON under `tuning_v2/`.

## tuning_v2/scripts/compile_sweeper.sh

Compiles all `.cu` variants from `tuning/sweeper_srcs/<dtype>/` into `tuning_v2/sweeper/<dtype>/bins/` (multi-variant sweeper binaries). Args: dtype (`e4m3|f16|all`), job count.

## .claude/skills/fsm-engine/scripts/pre-step-check.sh

CroqTuner FSM **precondition** gate for a named step; reads `CROQTUNER_STATE_FILE` (default `loop-state.json`); exits non-zero if step should not run (companion to agent tuning loop).

## .claude/skills/fsm-engine/scripts/post-step-check.sh

CroqTuner FSM **postcondition** gate after a step; ensures guard flags and artifacts are consistent.

## .claude/skills/fsm-engine/scripts/state-transition.sh

Atomically updates FSM state (INIT, transitions between BASELINE, PROFILE, IDEATE, IMPLEMENT, MEASURE, DECIDE, STORE, etc.); validates legal transitions; applies metric overrides.

## ralph-loop-request-data/s1/control_02.sh (experiment harness)

Standalone utility: find duplicate files in a directory by MD5 — not part of CrokTile product build.

## ralph-loop-request-data/s1/treatment_02.sh

Same class of harness script (duplicate-finder variant for A/B testing).

## scripts/snapshot.sh

Referenced by Makefile `snapshot` target — intended to snapshot paper state with a message; verify presence in your clone.
