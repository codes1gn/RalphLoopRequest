Summary: kernels/manifest.json schema (version 3)

Top-level fields
- version (number): schema/registry version (e.g. 3).
- description (string): human-readable summary of the registry purpose and coverage.

Hardware (object "hardware")
- gpu, sm, cuda, driver (strings): target GPU model, SM architecture tag, CUDA and driver versions.
- peak_f16_tflops, peak_f8_tflops (numbers): documented peak throughput.
- n_sm (number): SM count.
- smem_per_sm_bytes, l2_bytes (numbers): on-chip memory sizes for modeling/tuning context.

Repository paths (strings)
- choreo_repo, gemm_sp_paper_repo, gemm_sp_kernel_dir: absolute paths to related codebases and kernel directories.

Reference kernels (object "reference_kernels")
- Per dtype key (e.g. "f16", "e4m3"): object with best_cu (path), best_tflops, best_hw_eff, tuned_shape [M,N,K], origin_iter (number), origin_readme (path).

Baseline kernels (object "baseline_kernels")
- Optional "_note" (string).
- Per dtype: filename string for canonical baseline .co artifacts.

Scenarios (object "scenarios")
- Each scenario name maps to an object that includes:
  - label (string), type ("single" | "tied_pair").
  - For single: M, N, K may be numeric literals, the sweep token "S", or fixed dimensions; sizes (array of numbers) defines the sweep.
  - For tied_pair: tied_dims (array), free_dim (string), sub_series (object of named sub-scenarios), each with label, fixed (object of dims), and either tied_sizes or vary_sizes arrays.

Size constraints (object "size_constraints")
- min_M, min_N, min_K (numbers): validity bounds for shapes.

Region clustering (object "region_clustering")
- "_note" (string).
- near_threshold, far_threshold (strings): prose classification rules.
- typical_shapes_for_scratch (object): per-dtype arrays of [M,N,K] representative shapes (often "_fs" suffix in keys elsewhere).

Build (object "build")
- nvcc_flags, nvcc_includes, nvcc_libs (strings): default compile and link lines (with ${CHOREO_REPO} placeholders).
- f16_extra, e4m3_extra (strings): dtype-specific nvcc append flags.
- choreo_flags (string): choreo compiler invocation flags for from-scratch pipeline.
- warmup, iters (numbers): timing repeat configuration.

Verification (object "verification")
- Per dtype: base_tol, rel_tol (numbers) for numerical checks.

Tuning limits (object "tuning_limits")
- from_current_best, from_scratch: each has max_iter (number) and description (string).
