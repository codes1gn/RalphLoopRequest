CUDA kernels (.cu) and compiler flags used in this repo

Scope: Thousands of .cu files exist (kernel_bank trees, sweeper templates, variants). Flags are not embedded in .cu files; they come from build scripts, manifest.json, and skills. Below is a deduplicated list of compiler-related flags and options referenced for compiling or generating those kernels.

NVCC / CUDA (typical base — see kernels/manifest.json `build.nvcc_flags` and tuning scripts):
- -arch sm_90a  (alternate form in some scripts: -gencode arch=compute_90a,code=sm_90a)
- -std=c++17  (FSM skill example uses -std=c++20 for a direct nvcc one-liner — variant exists)
- -DCUTLASS_ENABLE_TENSOR_CORE_MMA=1
- -D__CHOREO_TARGET_CUTE__
- -D__USE_CUDA_TYPE__
- -D__CHOREO_DMA_DIAGNOSIS__
- -Xcompiler -static-libstdc++
- -O2 or -O3 or -O1  (sweeper variants / scripts differ; O1/O3 used as explicit experiment tags)
- --use_fast_math
- -ftz=true  (f16 dtype; e4m3 often omits per manifest `e4m3_extra`)
- Include paths: -I${CHOREO_REPO}/runtime -I${CHOREO_REPO}/extern/cutlass/include -I${CHOREO_REPO}
- Link: -L/usr/local/cuda/lib64 -lcuda
- Shape macros (compile-time): -DSPMM_DEFAULT_M=… -DSPMM_DEFAULT_N=… -DSPMM_DEFAULT_K=…
- Dtype: -DSPMM_DTYPE_E4M3 (e4m3 builds)

Choreo compiler (from-scratch / codegen path — manifest `build.choreo_flags` and skills):
- -gs -t cute -arch=sm_90a --use-warpspec --use-prepack
- (generate_f16_bank.py also shows variants like: choreo -t cute -arch=sm_90a -es … for expansion)

Per-shape / experiment “extra” flags:
- meta.json field extra_nvcc_flags (often empty string; can carry sweeper-applied deltas such as -O1 / -O3 when variant name implies opt level)

Environment (runtime, not nvcc flags but paired with binaries):
- CHOREO_TIMING_WARMUP, CHOREO_TIMING_REPEAT, CHOREO_SKIP_VERIFY

Source references: kernels/manifest.json (build), tuning_v2/scripts/*.py, tuning/kernel_bank/compile_all.sh, tuning_v2/scripts/compile_sweeper.sh, .claude/skills/fsm-engine/SKILL.md.
