Task 11 — Build configuration and build process

Root Makefile (`/home/albert/workspace/croktile_paper/Makefile`)
-----------------------------------------------------------------
Variables: `PAPER_DIR := paper`, `FIG_DIR := figures`.

Targets:
- `all` (default) → `paper`
- `svg2pdf` — For each `figures/out/*.svg`, if newer than corresponding `.pdf` or PDF missing, run `cairosvg` to produce PDF.
- `paper` — Depends on `svg2pdf`. Runs `cd paper && tectonic main.tex` to build the PDF. Then runs an inline Python snippet using `PyPDF2` to filter `paper/main.pdf` pages (drops near-empty or boilerplate pages based on extracted text regex); failures are ignored (`2>/dev/null || true`).
- `figures` — `cd figures && npm run build` (Node figure pipeline).
- `preview` — Depends on `paper`; removes `paper/preview/page-*.png` and runs `pdftoppm` on `main.pdf` for PNG previews.
- `clean` — Removes LaTeX auxiliary files under `paper/` and `figures/output/*`.
- `watch` — `cd paper && tectonic -X watch main.tex`.
- `snapshot` — `bash scripts/snapshot.sh "$(or $(MSG),draft)"` — **Note:** `scripts/snapshot.sh` was not present in the workspace at analysis time; this target would fail until that script exists or is added.

Implicit dependencies for a full paper build:
- `tectonic` (TeX engine)
- `cairosvg` (SVG→PDF)
- Python 3 with `PyPDF2`
- `pdftoppm` (optional, for `preview`)
- Node/npm in `figures/` with `npm install` for `d3`, `jsdom` per `figures/package.json` (`npm run build` → `node generate-all.js`)

CUDA / kernel builds
--------------------
There is no single top-level Makefile for CUDA kernels. Compilation is done via:
- Shell scripts under `tuning/kernel_bank/` and `tuning_v2/scripts/` invoking `nvcc` with `-I` to a `CHOREO_REPO` path (typically `/home/albert/workspace/choreo`), Cutlass includes, and `-L/usr/local/cuda/lib64`.
- Per-invocation flags from `meta.json` (`extra_nvcc_flags`) where applicable.

Summary
-------
**Paper:** `make` or `make paper` after figures assets are ready; use `make figures` first if SVG/JS figures must be regenerated. **Kernels:** use repo-specific `compile_*.sh` / Python drivers, not the root Makefile.
