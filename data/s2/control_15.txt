Task: README files and documentation completeness (control_15)

## README files found

- **No `README.md` at repository root** (as of this survey).
- **No `README` under `paper/`**.
- **figures/node_modules/** contains upstream READMEs from npm packages (d3, etc.) — not project documentation.

## Where documentation actually lives

1. **Cursor rules** (`.cursor/rules/`) — Substantive operational docs, e.g. `perf-tune.mdc` (tuning dispatcher), `tuning-workspace.mdc` (artifact layout, branch policy, iteration rules).

2. **Agent skills** (`.claude/skills/*/SKILL.md`) — Long-form procedures for tuning modes, FSM engine, push-marginal, iterative improvement, ralph-loop-request.

3. **FSM protocols** (`.claude/skills/fsm-engine/protocol/*.md`) — Identity, loop contract, checklists, compaction.

4. **kernels/manifest.json** — Human-oriented `description` plus machine-readable scenario and hardware metadata.

5. **Inline script docstrings** — Many `tuning/*.py` files begin with usage, paths, and log locations.

6. **Experiment / meta docs** — `ralph-loop-request-data/experiment-design.md` and related files describe harnesses, not the CrokTile product for end users.

7. **LaTeX paper** — `paper/main.tex` and sources are the academic artifact; they do not replace a repo README for developers.

## Assessment of completeness

- **For AI/agent-driven tuning**: Strong — rules and skills duplicate and enforce workflows that a typical README would only summarize.
- **For a new human cloning the repo**: **Weak** — absence of a root README means no single place for prerequisites (Tectonic, CUDA, sibling `choreo` checkout, env vars), directory map, or “how to build the paper vs run a sweep.” Developers must read Makefile + `.cursor/rules` + skills.
- **For paper reproducibility**: Partially covered by Makefile and `figures/package.json`; GPU reproduction depends on external repos and hardware not summarized in one file.

**Bottom line:** Documentation is **fragmented but deep** in agent-oriented files; **traditional onboarding README completeness is low.**
