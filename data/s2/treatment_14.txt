Task 14 — Python imports and dependency graph

Scope
-----
Analysis used AST import scanning over `.py` files under the repo, **excluding** `ralph-loop-request-data/` (A/B harness). Approximately **40** Python modules in `tuning/`, `tuning_v2/`, `tuning_v3/`, and `scripts/`.

Standard library (dominant)
---------------------------
Almost every script uses: `json`, `os`, `sys`, `subprocess`, `pathlib.Path`, `argparse` (where CLI), `re`, `time`, `shutil`, `math`, `tempfile`, `dataclasses`, `collections` (e.g. `Counter`), `concurrent.futures` for parallelism, `threading`/`queue` in one driver, `datetime` for timestamps, `importlib`/`unittest`/`hashlib` in `test_sweeper_isolation.py`.

Third-party libraries
---------------------
- **matplotlib**, **numpy**: `tuning_v3/plot_options.py`, `plot_comparison.py` (plotting pipeline).
- No widespread `pandas`; data is TSV/JSON-driven.

Internal / same-package imports
-------------------------------
- `tuning/best_suite/resweep_best.py` imports `from run_best_suite import compile_and_measure` — **same-directory** coupling to `run_best_suite.py` (must be run with cwd or PYTHONPATH including `tuning/best_suite/`).
- Otherwise modules are **standalone scripts**: no packaged `croktile_paper` top-level package; no `pip install -e .` layout detected in this repo slice.

Dependency graph (high level)
-----------------------------
```
scripts/generate_shape_suite.py, generate_state.py
  → json, collections — read/write manifest/state helpers

tuning/*.py, tuning_v2/**/*.py
  → stdlib + subprocess → external: nvcc, choreo binaries (not Python deps)
  → pathlib to repo-relative paths (logs, srcs, kernel_bank)

tuning_v3/*.py
  → json, os, csv + matplotlib, numpy — isolated plotting/data collection
```

Flow of data (not import edges): Python **writes** JSON/TSV consumed by other tools; **no** heavy internal import graph between `tuning` and `tuning_v2` — they are parallel experiment generations sharing similar concepts.

Execution environment
---------------------
Runtime deps for a minimal workflow: **Python 3.10+** (assumed), optional **matplotlib/numpy** for `tuning_v3`. GPU work requires CUDA toolkit at `/usr/local/cuda` and a checkout of **choreo** at the path expected by shell scripts (see hardcoded path analysis).
