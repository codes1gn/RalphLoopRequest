CrokTile kernels/manifest.json (v3) — structure summary

Top-level keys:
- version, description — schema version and human-readable purpose (kernel registry per dtype/scenario/shape; region clustering for seed selection).

- hardware — GPU environment: model, SM arch, CUDA/driver versions, peak TFLOPS (f16/f8), SM count, shared memory per SM, L2 size.

- choreo_repo, gemm_sp_paper_repo, gemm_sp_kernel_dir — absolute paths to related repos and the GEMM-SP kernel directory.

- reference_kernels — per dtype (f16, e4m3): path to the best tuned .cu, peak TFLOPS, hardware efficiency %, tuned shape [M,N,K], tuning iteration and README paths.

- baseline_kernels — canonical baseline .co filenames per dtype (from-scratch tuning starting point).

- scenarios — seven named benchmark scenarios (square, sweep_m, sweep_n, sweep_k, sweep_mn, sweep_mk, sweep_nk). Each has labels, dimension pattern (single-axis sweep with "S" or tied_pair with sub_series), and size lists defining which (M,N,K) shapes belong to the scenario.

- size_constraints — minimum M, N, K allowed.

- region_clustering — notes and thresholds classifying shapes as "near" or "far" relative to reference (4096,8192,8192); typical_shapes_for_scratch lists representative [M,N,K] triples per dtype for from-scratch runs (_fs artifact suffix).

- build — nvcc/choreo compile flags, includes, libs, dtype-specific extras, warmup and benchmark iteration counts.

- verification — per-dtype absolute and relative tolerance for numerical checks.

- tuning_limits — max iterations and descriptions for from_current_best vs from_scratch tuning modes.
