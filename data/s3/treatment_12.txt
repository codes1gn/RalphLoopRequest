Environment variable references in the croktile_paper codebase (human-readable summary; excludes third-party vendored JS inside large trees where noted).

== Build / tooling (manifest & shell) ==
- CHOREO_REPO — Referenced in `kernels/manifest.json` as `${CHOREO_REPO}` inside `build.nvcc_includes` include paths for nvcc. Must point to the choreo repository root at compile time.
- PATH — `tuning_v2/scripts/f16_run_and_update.py` prepends `/usr/local/cuda/bin` when spawning subprocesses: `os.environ['PATH']`.
- CUDA_VISIBLE_DEVICES — Same script sets GPU selection in the environment dict passed to subprocess.

== C++ / CUDA runtime (std::getenv) ==
Used in generated F16 kernel sources under `tuning_v2/` (templates, kernel_bank, sweeper variants). Variable names are string literals passed to `std::getenv`:
- CHOREO_DISABLE_TIMING — Optional; disables timing when set.
- CHOREO_SKIP_VERIFY — Optional; skips verification when set.
- CHOREO_TIMING_WARMUP — Optional; parsed as integer for warmup iterations.
- CHOREO_TIMING_REPEAT — Optional; parsed as integer for repeat count.

These four appear together in the same block across many `.cu` files (duplicated template bodies).

== Agent skills (FSM / CroqTuner) ==
Documented in `.claude/skills/fsm-engine/SKILL.md` and related skills:
- CROQTUNER_STATE_FILE — Selects which JSON state file the FSM scripts use (`loop-state.json`, `loop-state_from_scratch.json`, `iterative-improve-state.json`, or `push-marginal-state.json` depending on mode).

== Not environment variables ==
- `__CHOREO_*` tokens in `.cu` files are preprocessor macros / compile-time symbols, not OS environment variables.
- `CUDA_SUCCESS` in CUDA API checks is an enum constant, not an env var.

== Optional / external (if present in repo metadata) ==
- Husky git hooks may reference `$HUSKY_GIT_PARAMS` in package metadata in some setups; verify in `package.json` if using Husky locally.

Scan note: `ralph-loop-request-data/` may duplicate snippets from `kernels/manifest.json`; the authoritative CHOREO_REPO reference is `kernels/manifest.json`.
