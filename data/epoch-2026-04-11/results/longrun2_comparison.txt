Longrun2 — sA treatment vs control (sample: treatment_01.py vs control_01.py)

Both files solve the same problem: reverse a singly linked list in place in O(n) time and O(1) extra space by iteratively rewiring next pointers.

Treatment approach (treatment_01.py):
- Uses a top-level module docstring and modern PEP 604 unions (ListNode | None).
- Names the constructor parameter next_ to avoid colliding with the attribute name next and common builtins confusion.
- Types node values as object for generality.
- Names the public API reverse_linked_list (short, task-oriented name).

Control approach (control_01.py):
- Enables postponed evaluation of annotations via from __future__ import annotations and uses typing.Optional throughout.
- Uses int default for ListNode.val and Optional[ListNode] in signatures.
- Names the algorithm reverse_linked_list_in_place (more explicit about mutating behavior).
- Uses curr as the loop variable name (control) vs current (treatment); behavior is identical.

Conclusion: Algorithmically equivalent three-pointer walk; differences are naming, typing style (Optional vs |), and small API/documentation conventions rather than logic.
