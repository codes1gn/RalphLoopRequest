2) Treatment vs control (paired sample)
- Read treatment_01.py and control_01.py: both implement in-place linked list reversal with the same iterative pointer-rewiring pattern.
- Treatment favors PEP 604 unions, next_ constructor parameter, object-typed values, and the name reverse_linked_list.

- Detailed comparison: see longrun2_comparison.txt in this directory.

3) sB text artifacts
- 10 .txt files under data/epoch-2026-04-11/sB/ (paired treatment/control for five runs plus matching naming pattern).

4) statistics.md (results/statistics.md) — key findings
- n=40 subagents (20 control, 20 treatment); 100% task completion both arms.
- Treatment dramatically increased structured continuation: offered continuation 20/20 vs 1/20 control (p < 0.0001); contextual options 100% treatment vs 5% control.
- AskQuestion usage remained 0% in both arms (tool not available in environment).
- Verbose fallback: self-reported low (3/20 treatment) but manual inspection found explicit [durable-request]-style messaging in ~85% of treatment outputs — self-report unreliable.
- One control (B_03) was contaminated by reading SKILL.md and mimicking checkpoints; effect isolated to skill-reading tasks.
- Effect sizes for continuation rate are very large (Cohen's h ~2.69 overall; ~maximum if excluding contamination).
- Cross-epoch comparison: treatment checkpoint behavior stable vs epoch 2026-04-07; updated skill did not harm completion.

5) Synthesis
- Code side (sA): paired files are largely stylistic/typing variants on the same algorithms; line count (~1580 across 20 files) reflects small, focused solutions per pair.
- Experiment side: the durable-request skill v2 changes strongly shape end-of-run checkpoint and option-offering behavior without hurting task success; metrics around verbose fallback should trust transcript inspection over agent self-labels.
